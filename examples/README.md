# `examples/` — reference artifacts and demo state

What lives here:

| Subdir / file | What it is | Editable? |
|---|---|:---:|
| `*_default.soul.md` + `*_default.constitution.yaml` | One pair per original archetype role — the canonical example agent of each kind | regenerated, **don't hand-edit** |
| `lineage_*` | A 3-generation lineage demo (parent → child → grandchild) | regenerated |
| `stress_*` | Edge-case profiles (multiple flags, contradictory certainty) | regenerated |
| `audit_chain.jsonl` | The canonical audit chain — used by demo scripts and live-tests | append-only by daemon |
| `constitutions/` | Earlier-format constitution snapshots (legacy) | reference only |
| `skills/` | 26 skill manifests shipped as the v0.1 skill catalog | YAML — see below |

## What each pair shows

Every `<name>.soul.md` has a sibling `<name>.constitution.yaml`. The
soul is the LLM-rendered narrative voice + identity frontmatter; the
constitution is the machine-readable rulebook compiled from the same
trait profile. Both share the same content-addressed DNA (look at the
`dna:` field in the soul frontmatter and the `dna:` field at the top
of the constitution — they're identical, by design).

## Are they regenerated?

The `*_default.soul.md` and `lineage_*` and `stress_*` files are
produced by `scripts/demo_generate_soul.py`. The script:

1. Builds the trait profile for each archetype role
2. Generates the soul.md + constitution.yaml pair byte-for-byte
3. Appends the corresponding `agent_created` / `agent_spawned` events
   to `audit_chain.jsonl`
4. Verifies the chain end-to-end before exit

To regenerate after a trait-tree or constitution-template change:

```bash
python3 scripts/demo_generate_soul.py
```

Run from repo root. It writes to this directory — operators editing
soul.md by hand will lose edits next regeneration.

## `skills/` — the shipped skill catalog

26 YAML manifests covering:

- **Chain skills** (4) — the core Security Swarm escalation chain:
  `signal_match.v1` → `triage_signal.v1` → `contain_incident.v1` →
  `key_audit.v1`
- **Supporting skills** (17) — single-tool skills the swarm uses
  individually: log scans, baseline refreshes, USB audits, etc.
- **Triune meta-skills** (5) — SW-track + cross-triune ceremonies

Each manifest declares the tools it uses in `requires:`; the daemon
refuses to install a skill if any required tool isn't in the loaded
catalog. See `docs/runbooks/forge-tool-skill.md` for the
`fsf install skill` workflow.

## `audit_chain.jsonl` — the live audit chain

Append-only hash-linked JSONL. The default daemon writes here (per
`DaemonSettings.audit_chain_path`). Each line is one event:

```jsonl
{"seq": 0, "timestamp": "...", "prev_hash": "GENESIS", "entry_hash": "...", "agent_dna": null, "event_type": "chain_created", "event_data": {}}
{"seq": 1, "timestamp": "...", "prev_hash": "...", "entry_hash": "...", "agent_dna": "...", "event_type": "agent_created", "event_data": {...}}
```

Verify the chain at any time:

```bash
python3 scripts/verify_audit_chain.py
```

The chain is the source of truth — the SQLite registry is rebuildable
from it. Don't hand-edit this file; it'll fail verification. To
reset for a clean demo, use `./reset.command` which archives the
current chain to `<filename>.bak.<timestamp>` and starts fresh.

## Running examples in your own daemon

The `examples/audit_chain.jsonl` is what the daemon reads on startup
when `FSF_AUDIT_CHAIN_PATH` is unset. Demo scripts and live-test
commands use this path.

For an isolated demo without polluting your real state, use
`./scenarios/load-scenario.command synthetic-incident demo` — it loads
the canonical synthetic-incident scenario into the `demo/` directory
and `start-demo.command` points the daemon there.

## Don't-touch list

- `examples/audit_chain.jsonl` — append-only by daemon; manual edits
  break verification
- `examples/*_default.*` — regenerated by the demo script
- `examples/lineage_*` — same
- `examples/stress_*` — same

## Free to edit

- `examples/skills/*.yaml` — these ARE the live skill catalog the
  daemon reads at lifespan. Edit one, then `POST /skills/reload`
  (or restart the daemon) to pick up the change.
- This `README.md` — orientation only
